Приложение “Электронный блокнот”. Часть 1.

1.     Спроектируйте классы-сущности системы:  Note,  NoteBook.
Переопределите для этих классов методы equals, hashCode и toString().
Напишите тесты, позволяющие проверить корректность работы экземпляров этих классов.


2.     Создайте класс NoteBookConsoleView, перегрузите в нем метод print так,
 чтобы иметь возможность выводить на консоль данные об одной записи,
  данные о нескольких записях или всю записную книжку (при перегрузке примените метод с переменным числом параметров).


3.     Создайте класс NoteBookProvider, реализующий шаблон Singleton(однопоточный), для доступа к экземпляру класса NoteBook.


Приложение “Электронный блокнот”. Часть 2.

Переопределите в классах Note, NoteBook методы equals(), hashCode(), toString().

Реализуйте следующую бизнес-логику работы приложения с использованием шаблона Command:
-          Создать Новый Блокнот?
-          Добавить запись в Блокнот
-          Найти записи в блокноте по содержимому
-          Найти записи в блокноте по дате создания
-          Показать записи в блокноте
-          Записать Блокнот в файл
-          Загрузить Блокнот из файла
Для доступа к сущности Блокнот(NoteBook) используйте класс NoteBookProvider.
Создайте классы Request и Response, позволяющие передавать и возвращать параметры.

Создайте тесты, проверяющие правильность работы вашей бизнес-логики.
